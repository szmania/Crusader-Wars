# Linux/Proton Setup Wizard - Implementation Plan (Crusader Conflicts)

## 1. Summary / Overview

### Problem Statement
The current Linux/Proton setup process for Crusader Conflicts requires 10+ manual steps including:
- Wine prefix creation and configuration
- .NET 4.7.2 installation via winetricks
- Mod symlinking for Attila
- Shortcut creation for Attila launcher
- Steam launch options configuration
- Proton compatibility settings

This complex multi-step process creates a high barrier to entry for Linux users.

### Solution
Create an in-app wizard that automates the entire setup process into a single-click operation, reducing user effort from 10+ manual steps to a guided wizard with minimal user input. The wizard will:
- Auto-detect Linux/Proton environment and launch automatically on first detection
- Provide a manual trigger button in the ModOptions screen
- Use the default Wine prefix: `~/.crusader-conflicts-net-pfx`
- Focus exclusively on Attila/Proton configuration (not CK3)
- Support GNOME and other widely-used desktop environments (KDE, XFCE, etc.)
- Display real-time status messages and errors to the user
- Log all operations to the existing file logging system

### Expected Outcomes
- ✅ Auto-detect Linux/Proton and launch wizard automatically on first run
- ✅ Manual trigger button in ModOptions screen (top-left, above game path settings)
- ✅ Automated Wine prefix setup with .NET 4.7.2 using `~/.crusader-conflicts-net-pfx`
- ✅ Automatic mod symlinking for Attila
- ✅ Shortcut creation and Steam configuration
- ✅ User-friendly progress tracking with real-time status messages
- ✅ Comprehensive file logging with user-visible error/status notifications
- ✅ Support for GNOME, KDE, XFCE, and other common desktop environments

## 2. Design Requirements

### 2.1 User Requirements
- **UR-001**: Wizard auto-launches on first Linux/Proton detection
- **UR-002**: Manual trigger button in ModOptions screen (top-left, above game path settings)
- **UR-003**: Clear step-by-step progress indication with real-time status messages
- **UR-004**: Minimal user input required (only game paths if not auto-detected)
- **UR-005**: Error messages with actionable solutions displayed to user
- **UR-006**: Adequate user confirmation before major system changes
- **UR-007**: Ability to re-run setup if needed

### 2.2 Software Requirements
- **SR-001**: Detect if running on Linux/Proton environment
- **SR-002**: Detect installed Wine version
- **SR-003**: Detect Steam installation and game paths
- **SR-004**: Execute shell commands via Wine/Proton
- **SR-005**: Create symlinks for Attila mods
- **SR-006**: Create Windows shortcuts (.lnk files)
- **SR-007**: Modify Steam launch options
- **SR-008**: Install .NET 4.7.2 via winetricks
- **SR-009**: Handle file system operations in Linux paths
- **SR-010**: Log all operations to existing file logging system
- **SR-011**: Display real-time status and errors to user during setup

### 2.3 Compatibility Requirements
- **CR-001**: Support for Steam and non-Steam game installations
- **CR-002**: Support for GNOME, KDE, XFCE, and other common desktop environments
- **CR-003**: Compatible with Proton 7.0+
- **CR-004**: Support for various Wine configurations
- **CR-005**: Handle different Linux distributions (Ubuntu, Fedora, Arch, etc.)

### 2.4 Performance Requirements
- **PR-001**: Setup should complete within 5-10 minutes (mostly .NET installation)
- **PR-002**: Non-blocking UI with progress updates
- **PR-003**: Minimal disk space usage for temporary files

## 8. Technical Notes

### 8.1 Path Translation
Wine uses Z:\ to represent the Linux root (/). The wizard must handle path conversions:
- Linux: `/home/user/Games/SteamGames/Total War Attila`
- Wine: `Z:\home\user\Games\SteamGames\Total War Attila`

### 8.2 Command Execution
All shell commands must be executed through Wine:
```bash
WINEPREFIX=~/.crusader-conflicts-net-pfx wine [command]
```

### 8.3 Symlink Considerations
- Symlinks should use absolute paths
- Check for existing files before creating symlinks
- Handle permission errors gracefully

### 8.4 Steam Launch Options
Example launch options for Attila:
```
PULSE_LATENCY_MSEC=60 PROTON_ENABLE_WAYLAND=1 mangohud gamemoderun %command% used_mods_cw.txt
```

### 8.5 Desktop Environment Detection
Detect desktop environment using:
- `echo $XDG_CURRENT_DESKTOP`
- `echo $DESKTOP_SESSION`
- Check for specific process names (gnome-shell, kwin, xfce4-session, etc.)

### 8.6 Settings Integration
Store configuration in `CrusaderWars/Properties/Settings.settings`:
- Add new settings for Linux setup configuration
- Track setup completion status
- Store game paths and preferences
